# CallBrain ‚ÄĒ Phased Build Plan

> Companion to `ARCHITECTURE.md`. **Path-B oriented:** the core capture‚Üíindex‚Üíask loop works at Phase 1; everything after is depth + polish. **Every phase ends with a Codex audit gate** ‚ÄĒ `codex exec -s read-only` over the branch diff against a written checklist (a second pair of eyes, per the founder's standing rule). Workstreams marked **‚ą•** are parallelizable within a phase. Each phase ships behind a feature branch and is only "done" after its Codex gate is green and any HIGH findings are fixed.

**Audit protocol (every phase):** build green ‚Üí eval/tests green ‚Üí `codex exec -s read-only` review of the diff + the phase checklist ‚Üí fix HIGH/CRITICAL ‚Üí re-review if needed ‚Üí mark phase complete in the ledger (¬ßLedger) ‚Üí memory note updated.

## Phase 0 ‚ÄĒ Foundations & Ground-Truth Verification
- **Goal:** stand up the Swift project + the custom SQLite, and prove every external assumption against *real* artifacts before building on it.
- **Deliverables:**
  - **‚ą•A** Xcode project (macOS 26, Swift 6 strict concurrency) + SPM deps wired: **GRDB** (custom SQLite build), **WhisperKit**, **FluidAudio**, **swift-embeddings**, **swift-subprocess**, **Sparkle**, (usearch deferred). CI + a **grep-gate** banning `--bare`, `--dangerously-*`, `ANTHROPIC_API_KEY=`.
  - **‚ą•B** Custom SQLite build proven: FTS5 **and** sqlite-vec compiled into ONE library (`SQLITE_CORE`, static), opens via GRDB, a `vec0` table accepts a 768-float vector + metadata columns and does a KNN query. (Verify gate: ¬ß15 custom-build + notarization.)
  - **‚ą•C** **CLI capability probe** harness: run the ¬ß5 micro-calls for `claude` 2.1.196 and `codex` 0.142.3, snapshot the JSON envelope shapes, confirm env-scrub forces subscription auth, confirm `--safe-mode --tools ""` answers cleanly + returns schema JSON.
  - **‚ą•D** Embedding bring-up: nomic-embed-text-v1.5 returns a 768-vector for a doc + a query (in-process CoreML target; ollama fallback measured). Confirm query/doc prefixes.
  - **‚ą•E** **Collect ~5 real samples of each source** (Fathom copy, Fireflies JSON, Gemini/Meet Doc, Cluely note, raw Meet `.mp4`); snapshot format fingerprints; verify whether the founder's Drive `Meet Recordings` has sibling Transcript Docs vs Notes-only.
- **Dependencies:** none.
- **Exit:** one real artifact of each available type round-trips through a throwaway parser into the CTM; `claude -p` and `codex exec` each return a clean answer + valid schema JSON; a 768-vector KNN round-trips through the custom-SQLite `vec0` table.
- **Codex gate:** env-isolation correctness (API-key scrub, empty sandbox), the verified-vs-assumed table, the custom-SQLite build flags, grep-gate coverage.

## Phase 1 ‚ÄĒ Core Capture‚ÜíIndex‚ÜíAsk Loop (MVP, usable this week)
- **Goal:** drop a transcript ‚Üí ask a cited question. The loop genuinely works.
- **Deliverables:**
  - **‚ą•A** Parsers for the 2 formats the founder has most (**Fireflies JSON + Fathom copy**) ‚Üí CTM normalize ‚Üí speaker-turn **chunker** (512/128/768).
  - **‚ą•B** **CallBrainDB** (GRDB): the canonical schema (¬ß8) + FTS5 triggers + the `vec_chunks__nomic__v1` `vec0` table; write path (relational + FTS5 + vector).
  - **‚ą•B** **SearchEngine**: hybrid retrieval ‚ÄĒ FTS5 BM25 ‚äē vector, **selectivity-routed (D6)** exact path, RRF (k=60), refusal/weak gates, citation validator.
  - **‚ą•C** **EmbeddingActor** (nomic in-process) + **LLMRunner Claude adapter** (`complete` + `complete_json` + streaming) with env-scrub + injection-inert flags.
  - **‚ą•D** Minimal SwiftUI shell: **Ask AI** (streaming + tappable citations), **Meetings** list, **Meeting Detail**, **Transcript Viewer** (jump-to-citation). **General Ask + Person** modes with the full citation envelope + real refusal on no-evidence.
- **Dependencies:** Phase 0.
- **Exit:** "What did Travis say about Render?" ‚Üí correct cited answer with tap-to-jump anchors; a no-evidence question **refuses**; "compute provider" (semantic) surfaces Render/OpenRouter chunks; exact "Render" (keyword) ranks literal chunks #1.
- **Codex gate:** citation enforcement (no claim without a chunk), pre-filter correctness (date/speaker actually applied ‚ÄĒ the D6 exact path returns all in-scope golds), refusal-before-generation, actor isolation (no `@MainActor` blocking on DB/vector/subprocess).

## Phase 2 ‚ÄĒ Ingestion Intelligence & Durable Pipeline
- **Goal:** "just detect and do the right thing" for every source, idempotently, never silently.
- **Deliverables:** full **3-stage detector** + routing table + Meet sibling-pairing; all remaining parsers (Gemini Doc, Cluely, SRT/VTT, generic) tolerant + per-file confidence + **fingerprint learning**; metadata auto-heal + filename normalization; hybrid **entity/NER** (NaturalLanguage + gazetteer + LLM-assist); two-tier BLAKE3 idempotency + duplicate-group detection; the full **import state machine** with per-state checkpoints, **durable GRDB job queue**, never-silent-fail wrapper; **Import Queue + needs_review UI**; live progress stream.
- **‚ą•:** (A) detector+router+pairing ¬∑ (B) remaining parsers+fingerprints ¬∑ (C) state machine+queue+resumability ¬∑ (D) entities/NER ¬∑ (E) Import Queue/needs_review UI.
- **Dependencies:** Phase 1.
- **Exit:** dropping a mixed folder routes each file correctly or parks it with a plain-English reason; crash mid-import resumes from the last checkpoint; re-dropping a file is a no-op.
- **Codex gate:** confidence-gate math (no coin-flip routing), idempotency (no dup meetings, no re-transcribe), state-machine resumability, every exception path ‚Üí `failed`/`needs_review` (no silent drop).

## Phase 3 ‚ÄĒ Local Transcription Path (raw Google Meet video)
- **Goal:** a raw `.mp4` with no transcript becomes a first-class, diarized, cited meeting ‚ÄĒ on-device, no torch, credits saved.
- **Deliverables:** AVFoundation ‚Üí 16 kHz mono; **WhisperKit `large-v3-turbo`** (word ts, VAD) + **FluidAudio** diarization + **midpoint word‚ÜĒturn alignment** + churn smoothing; `transcript_versions` (local v0 immutable); per-file **"upgrade to cloud transcription"** (Deepgram/AssemblyAI); transcription progress as fraction-of-audio; first-run model download + signature-verify; Apple SpeechTranscriber live/bridge option.
- **‚ą•:** (A) decode+WhisperKit ¬∑ (B) FluidAudio+alignment ¬∑ (C) model-asset download/verify ¬∑ (D) cloud-upgrade hook + version UI.
- **Dependencies:** Phase 2.
- **Exit:** a real raw Meet recording ‚Üí speaker-labeled, timestamped, citable transcript; cloud upgrade appends v1 without destroying v0; citations re-derive from the active version.
- **Codex gate:** diarization-alignment correctness, `is_inferred_speaker` propagation into citations, model signature verification before use, ANE/CPU resource caps (no UI starvation). **Verify gates: WhisperKit WER + FluidAudio DER on real crypto calls.**

## Phase 4.5 ‚ÄĒ Fireflies-Grade Meeting Workspace & Conversational Intelligence
- **Goal:** the *open-a-meeting* and *ask-AI* experiences match Fireflies' polish ‚ÄĒ a tabbed **Notes | Transcript** meeting workspace with a persistent **AskFred-style** chat, **timestamp-linked citations** that jump into a clean readable transcript, transparent **agentic reasoning** shown live during a search, and durable, browsable **conversation history (Recents)**. *(Founder direction 2026-06-30, with 3 reference screenshots ‚ÄĒ see `DESIGN-fireflies-reference.md` ¬ß"Meeting workspace & AskFred".)*
- **Deliverables:**
  - **‚ą•A Meeting workspace shell** ‚ÄĒ a two-pane meeting view: left = content pane with **Notes | Transcript** segmented tabs (room for an "AI Skills/Actions" tab); right = a persistent **AskFred chat docked to this meeting** (ask *within this call*) with its own quick **Transcript** tab. Expand/focus toggle, clean header (title ¬∑ owner ¬∑ date ¬∑ language).
  - **‚ą•A Notes view** ‚ÄĒ structured AI summary: **section headers ‚Üí bullets ‚Üí sub-bullets**, each anchored to a **clickable `[MM:SS]` timestamp** that scrolls + flashes the matching transcript turn. A **"General Summary ‚Ėĺ / Refine Summary"** control (regenerate / condense / expand, cited). Gemini-notes meetings reuse `GeminiNotesView`; transcript meetings get a **cited LLM summary pass** (deterministic section seeds + summarizer, every bullet carries a timestamp anchor ‚ÄĒ no uncited claim).
  - **‚ą•B Clean Transcript reader** ‚ÄĒ the raw transcript rendered Fireflies-clean: speaker avatar + name + clickable timestamp + well-spaced paragraph per turn; a **Find bar** (in-transcript search w/ match highlight + next/prev); jump-to-timestamp; **deep-link target** for chat citations (tap `[03:50]` ‚Üí scroll + flash the 03:50 turn); a **"Sync with audio"** affordance wired when Phase-3 media exists (hidden otherwise).
  - **‚ą•C Timestamp-linked citation model** ‚ÄĒ unify `Citation` to carry `meetingID + tStart + speaker`, so Ask answers cite **`[MM:SS]`** anchors that are clickable and navigate into the right meeting's transcript at the right turn (across-meeting and within-meeting).
  - **‚ą•D Agentic, transparent AI search** ‚ÄĒ the **reasoning timeline** UX: during a search, stream the *real* pipeline as labeled steps ‚ÄĒ **Understanding query ‚Üí Selecting mode ‚Üí Finding meetings ("Searching for 'rendering'‚Ä¶") ‚Üí Analyzing ‚Üí Answer** ‚ÄĒ inside a collapsible **"Thinking Ns"** disclosure, then the streamed cited answer. Steps reflect actual Phase-4 planner/retrieval work (intent + mode + filters + retrieval + synthesis) ‚ÄĒ **never fabricated theater**; a step is shown only when that work runs.
  - **‚ą•E Conversation history / Recents** ‚ÄĒ persistent chat **sessions**: durable `conversations` + `messages` schema; a left-rail **Recents** list (auto-named from the first question + timestamp + snippet) to revisit/branch prior chats; per-conversation follow-up context; rename/delete; search across past chats. Applies to **both** the global Ask AI surface and per-meeting AskFred.
- **‚ą•:** (A) workspace shell + Notes ¬∑ (B) transcript reader + find/jump ¬∑ (C) timestamp citation model ¬∑ (D) agentic reasoning stream ¬∑ (E) conversation persistence + Recents.
- **Dependencies:** Phase 1 (ask loop), Phase 4 (planner/modes/action-items power the cited notes + the reasoning steps). **Sequencing:** runs after Phase 4; ‚ą•A/‚ą•B (workspace + transcript reader) may start earlier as they only need Phase-1 data. Packaging (Phase 8) remains the final ship gate.
- **Exit:** opening a meeting shows **Notes** (sectioned, `[MM:SS]`-linked) and a clean **Transcript** with working Find; a chat answer's `[03:50]` citation jumps into the transcript and flashes that turn; an AI search streams a labeled reasoning timeline then a cited streamed answer; quitting/reopening Ask shows prior chats in **Recents** and reopening one restores the full thread.
- **Codex gate:** citation‚Üíturn anchor correctness (timestamp maps to the right turn, no off-by-one), **reasoning-step truthfulness** (every displayed step ‚ÜĒ real pipeline work), conversation-persistence integrity (no message loss/reorder, FK cascade on delete), long-transcript render performance (lazy, no main-thread stalls), Swift-6 isolation of the streaming reasoning + chat-session state.
